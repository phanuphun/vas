การตั้งค่าหลัง Login (.xprofile)

{{ 'ตั้งค่าแล้ว' if display_config.has_signature else 'ยังไม่ได้ตั้งค่า' }}

เขียนสคริปต์ display-session.sh แล้วเรียกจาก ~/.xprofile ของเฉพาะ user ที่ login อยู่ตอนกด Apply เท่านั้น — ทำงานหลัง login (รอ 5 วินาที แล้วเช็คซ้ำสูงสุด 30 ครั้งว่าจอ/ทัชพร้อมหรือยัง) ก่อนจะหมุนจอ/ปรับ touch ให้จริง ใช้เมื่อต้องการตั้งค่าเฉพาะ user คนใดคนหนึ่งบนเครื่องที่มีหลาย user (เช่น ค่าของ kiosk-user ต่างจาก hapymed)

พบว่าถ้าเปิด toggle นี้พร้อมกับ "Persist touch ใน Xorg" ในรอบเดียวกัน touch อาจเพี้ยนได้ — เป็นคนละ mechanism ที่เขียนทับ Coordinate Transformation Matrix คนละจังหวะเวลา (ตัวนี้เขียนตอน login, อีกตัวเขียนตอน X server เริ่มก่อน login) จึงตั้งเป็น optional ต้อง toggle เองเสมอ ไม่มี default เปิดให้ แนะนำให้ทดสอบทีละอย่าง เปิดตัวเดียว reboot เช็คผลก่อน ค่อยเปิดอีกตัว
สร้าง .xprofile + display-session.sh (Persist at login)

Toggle เป็น ON อัตโนมัติถ้าเครื่องมีไฟล์นี้ตั้งไว้แล้วจริง — เปิด toggle = เขียนไฟล์ทันที, ปิด toggle = ลบไฟล์ทันที โดยใช้ค่าจอ/ทัช/ทิศทางที่เลือกไว้ในการ์ด Monitor Setting ด้านบน ณ ขณะนั้น ไม่ต้องรอกด Apply

ตั้งค่าระดับเครื่อง (Xorg — ก่อน Login)

ไฟล์ในหมวดนี้ถูก X server อ่านตอนเริ่มทำงานครั้งแรก ก่อนมีใคร login เลย (ก่อนหน้าจอ GDM greeter จะขึ้นด้วยซ้ำ) มีผลกับทุก user บนเครื่องเหมือนกันหมด ไม่ผูกกับ home ของใครคนใดคนหนึ่ง และไม่มีการรอ/เช็คซ้ำแบบ script ต่อ user ด้านบน

ใช้ได้ผลดีที่สุดกับ session ที่ไม่มีตัวจัดการจอของตัวเอง เช่น Openbox (แนะนำให้ใช้กับ kiosk-user) — session แบบ GNOME (เช่น GDM greeter หรือ user ที่ตั้ง session type เป็น gnome) มี GNOME Shell/mutter คอยเขียนทับค่านี้กลับเป็น normal อีกทีตอน compositor เริ่มทำงาน ถ้าเป็นกรณีนี้ยังต้องพึ่ง ".xprofile" การ์ดด้านบนแทนอยู่ดี

monitors.xml (GNOME/mutter — ระดับเครื่อง)

{{ 'ตั้งค่าแล้ว' if monitors_xml.has_signature else 'ยังไม่ได้ตั้งค่า' }}

แก้ปัญหาที่ session แบบ GNOME (mutter) เขียนทับค่าหมุนจอกลับเป็น normal เสมอ ถ้า user ที่ login ไม่มี ~/.config/monitors.xml ของตัวเอง — VAS เรียก D-Bus ของ mutter (session ที่ active อยู่ตอนนี้) เพื่อ query ค่าฮาร์ดแวร์จอจริง แล้วเขียนไฟล์ระดับเครื่อง /etc/xdg/monitors.xml ให้เอง — ไม่ต้องเข้า GNOME Settings บนจอเครื่องจริงเลย
