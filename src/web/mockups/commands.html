คู่มือคำสั่ง VAS CLI

รวมคำสั่งที่ใช้บนเครื่อง vending machine จัดกลุ่มตามหน้าที่ — คัดลอกไปรันใน terminal ได้โดยตรง

หน้านี้แสดงคำสั่งเพื่ออ้างอิงเท่านั้น — รันด้วยตนเองใน terminal บนเครื่อง vending machine คำสั่งที่มีป้าย sudo ต้องใช้สิทธิ์ผู้ดูแลระบบ
สารบัญ
  • ติดตั้ง
  • ถอน / รีเซ็ต
  • จอแสดงผล
  • VPN
  • เว็บเซิร์ฟเวอร์

VAS CLI เป็นชุดคำสั่งสำหรับติดตั้ง จัดการ และแก้ไขเครื่อง vending โดยตรง แต่ละคำสั่งด้านล่างมีคำอธิบายสั้นๆ และตัวอย่างที่พร้อมคัดลอก หากต้องการควบคุมผ่าน UI ให้ใช้หน้า แดชบอร์ด หรือหน้าย่อยที่เกี่ยวข้องแทน

ติดตั้ง (Install)

6 คำสั่ง

ติดตั้งส่วนประกอบที่ VAS รองรับบนเครื่อง ใช้พารามิเตอร์ --component เลือกส่วนที่ต้องการ หรือใช้ all เพื่อติดตั้งทั้งหมดในครั้งเดียว

Install all sudo

ติดตั้งส่วนประกอบทั้งหมดที่ VAS จัดการได้ — git, node, docker, wireguard และ anydesk

sudo vas install --component all
Install git sudo

ติดตั้ง Git สำหรับดึงและอัปเดตซอร์สโค้ดบนเครื่อง

sudo vas install --component git
Install node sudo

ติดตั้ง Node.js สำหรับรันแอปพลิเคชัน frontend ของเครื่อง vending

sudo vas install --component node
Install docker sudo

ติดตั้ง Docker สำหรับรันบริการใน container

sudo vas install --component docker
Install wireguard sudo

ติดตั้ง WireGuard สำหรับเชื่อมต่อ VPN ไปยังเครือข่ายส่วนกลาง

sudo vas install --component wireguard
Install anydesk sudo

ติดตั้ง AnyDesk สำหรับ remote support และแก้ไขปัญหาจากระยะไกล

sudo vas install --component anydesk

จอแสดงผล (Display)

7 คำสั่ง

จัดการการหมุนจอ หน้าจอสัมผัส และโหมด Wayland ของ GDM คำสั่ง apply มีผลทันที ส่วน persist บันทึกให้คงอยู่หลังล็อกอินใหม่

Show display status

แสดงสถานะจอและ output ที่เชื่อมต่ออยู่บน display ที่ระบุ

vas display status --display :0
List touchscreens

แสดงรายการอุปกรณ์หน้าจอสัมผัสที่ระบบตรวจพบ

vas display list-touch --display :0
Disable GDM Waylandsudo

ปิด Wayland บน GDM ให้ใช้ Xorg แทน — ช่วยแก้ปัญหาความเข้ากันได้กับหน้าจอสัมผัสบางรุ่น

sudo vas display disable-wayland
Enable GDM Waylandsudo

เปิด Wayland บน GDM กลับมาใช้งาน

sudo vas display enable-wayland
Apply runtime

หมุนจอและจัดแมปหน้าจอสัมผัสทันที โดยไม่บันทึกถาวร — ใช้ทดสอบก่อน persist

vas display apply --display :0 --output Virtual1 --touch 'Vending Virtual Touchscreen' --rotate normal
Persist session

บันทึกการหมุนจอและ touch mapping ให้คงอยู่ใน session ปัจจุบันและครั้งถัดไปที่ล็อกอิน

vas display persist-session --display :0 --output Virtual1 --touch 'Vending Virtual Touchscreen' --rotate normal
Persist touch in Xorgsudo

บันทึกการตั้งค่าหน้าจอสัมผัสลงใน config ของ Xorg — ใช้เมื่อต้องการให้คงอยู่แม้ reboot

sudo vas display persist-xorg --touch 'Vending Virtual Touchscreen' --rotate normal

VPN — WireGuard

8 คำสั่ง

ติดตั้ง สร้าง ตรวจสอบ และ sync config WireGuard กับ systemd บนเครื่อง ดูรายละเอียดเพิ่มเติมได้ที่หน้า VPN

Installsudo

ติดตั้ง WireGuard ผ่าน VAS CLI

sudo vas wireguard install
Create template

สร้างไฟล์ config ต้นแบบ wg0.conf สำหรับแก้ไขค่า key และ endpoint

vas wireguard init-config --name wg0 --output ./wg0.conf
Validate config

ตรวจสอบความถูกต้องของไฟล์ config ก่อนนำไปใช้งานจริง

vas wireguard validate --config ./wg0.conf
Save config

บันทึก config ที่แก้ไขแล้วเข้าสู่ที่เก็บของ VAS

vas wireguard save --name wg0 --config ./wg0.conf
Sync configsudo

นำ config ที่บันทึกไว้ไปใช้กับ WireGuard บนระบบ (systemd)

sudo vas wireguard sync --name wg0
Show status

แสดงสถานะการเชื่อมต่อและ interface ของ WireGuard

vas wireguard status --name wg0
List history

แสดงประวัติ config ที่เคยบันทึกไว้

vas wireguard history --name wg0
Unsync configsudo

ยกเลิกการ sync config ออกจาก WireGuard บนระบบ

sudo vas wireguard unsync --name wg0

เว็บเซิร์ฟเวอร์ (Server)

4 คำสั่ง

ควบคุมบริการ VAS web UI ที่ให้เข้าถึงหน้าแดชบอร์ด จอแสดงผล VPN และบันทึกระบบผ่านเบราว์เซอร์

Start background servicesudo

เริ่ม VAS web server เป็นบริการพื้นหลัง ฟังที่พอร์ต 8888

sudo vas server start --host 0.0.0.0 --port 8888
Show service status

ตรวจสอบว่าเว็บเซิร์ฟเวอร์กำลังทำงานอยู่หรือไม่

vas server status
Run foreground

รันเว็บเซิร์ฟเวอร์ในโหมด foreground — เหมาะสำหรับ debug หรือทดสอบชั่วคราว

vas server run --host 0.0.0.0 --port 8888
Stop servicesudo

หยุดบริการ VAS web server

sudo vas server stop
