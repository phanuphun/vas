# คู่มือฉบับเข้าใจง่าย: ทำไมจอกับทัชสกรีนถึง "เพี้ยน" หลัง Restart และควรตั้งค่าตามลำดับไหน

> เขียนสำหรับคนที่ไม่ต้องรู้ Linux มาก่อน อธิบายด้วยการเปรียบเทียบ ไม่ใช้ศัพท์เทคนิคโดยไม่อธิบาย

---

## 1. สรุปสั้นๆ ก่อนว่าเกิดอะไรขึ้น

ทำตามลำดับนี้: Reset ค่าเดิมทิ้ง → หมุนจอเป็น Left (ทัชตรงดี) → เปิดปุ่ม "Persist touch in Xorg" เพิ่ม → Apply → รีสตาร์ทเครื่อง

ผลที่ได้: **จอกลับไปเป็น Normal เหมือนเดิม แต่ทัชยังคิดว่าจอเอียง Left อยู่** เลยกลายเป็นแตะจุดหนึ่งแล้วไปโดนอีกจุดหนึ่ง

สาเหตุคือ ระบบมี "ที่เก็บค่าการตั้งจอ" อยู่ 2 ที่ ที่**ไม่ผูกกัน**และ**ไม่ได้ถูกอ่านพร้อมกัน** — เปรียบเหมือนบ้านหลังหนึ่งมีป้ายติดหน้าบ้าน 1 ป้าย กับสมุดจดส่วนตัวของแต่ละคนในบ้านอีกคนละเล่ม ถ้าป้ายหน้าบ้านกับสมุดส่วนตัวเขียนไม่ตรงกัน จะสับสนได้ทันที รายละเอียดอยู่หัวข้อถัดไป

---

## 2. เปรียบเทียบง่ายๆ: มี "ที่เก็บค่า" 2 แบบ ไม่ใช่แบบเดียว

### แบบที่ 1 — "ป้ายหน้าบ้าน" (ไฟล์ตั้งค่า Touch ของ Xorg)

- ไฟล์จริง: `/etc/X11/xorg.conf.d/99-vending-touchscreen.conf`
- เปรียบเหมือน **ป้ายติดหน้าบ้านหลังเดียว ใช้ร่วมกันทุกคนที่อาศัยอยู่ในบ้าน**
- ป้ายนี้ถูกอ่าน**ตอนบ้านเปิดไฟทั้งหลังครั้งแรก** (คือตอน "หน้าจอกราฟิก" เริ่มทำงาน — เร็วกว่าตอนที่จะมีใครเดินเข้าบ้าน/login ด้วยซ้ำ)
- ป้ายนี้คุมแค่เรื่องเดียว: **ทิศทางการสัมผัสของทัชสกรีน (touch)** เท่านั้น
- ข้อดี: ไม่ต้องสนใจว่าใครจะ login เข้ามาใช้เครื่อง ป้ายนี้ใช้ได้กับทุกคน
- ข้อเสีย: ต้อง "ปิดไฟทั้งบ้านแล้วเปิดใหม่" (reboot หรือ logout เข้าใหม่) ป้ายถึงจะมีผล — แก้ป้ายแล้วไม่ reboot จะยังไม่เห็นผลอะไรเลย

### แบบที่ 2 — "สมุดจดส่วนตัว" (ไฟล์ `.xprofile` + `display-session.sh`)

- ไฟล์จริง: `~/.xprofile` และ `~/.config/vending-auto-setup/display-session.sh` (เครื่องหมาย `~` แปลว่า "โฟลเดอร์บ้านของคนคนนั้น" — แต่ละคนที่ login เข้าเครื่องได้ จะมีโฟลเดอร์บ้านเป็นของตัวเอง แยกจากคนอื่นเสมอ)
- เปรียบเหมือน **สมุดจดส่วนตัวของแต่ละคนในบ้าน** — คนละเล่ม คนละลิ้นชัก ไม่มีใครเห็นสมุดของคนอื่น
- สมุดเล่มนี้ถูกเปิดอ่าน**เฉพาะตอนที่เจ้าของสมุดคนนั้นเดินเข้าบ้าน (login) เท่านั้น** ถ้าเป็นคนอื่น login สมุดเล่มนี้จะไม่ถูกเปิดเลย
- สมุดเล่มนี้คุม 2 เรื่องพร้อมกัน: **ทิศทางการหมุนจอ (rotate)** และ **ทิศทางการสัมผัส (touch)**
- สมุดนี้ไม่ได้ถูกอ่านทันทีที่เข้าบ้าน แต่จะ**รอ 5 วินาทีก่อน แล้วค่อยๆ เช็คซ้ำทุก 1 วินาที นานสุด 30 ครั้ง** ว่าจอ/ทัชพร้อมหรือยัง (เพราะบางทีจอกับทัชสกรีนยังไม่ปรากฏตัวใน 1-2 วินาทีแรกหลัง boot) — **ถ้าเช็คจนครบ 30 ครั้งแล้วยังไม่เจอ สมุดนี้จะ "หยุดอ่านเงียบๆ" ไม่มีอะไรแจ้งเตือนบนหน้าเว็บเลย**

### จุดสำคัญที่สุด: **ไม่มี "ป้ายหน้าบ้าน" สำหรับการหมุนจอเลย**

ป้ายหน้าบ้าน (แบบที่ 1) มีแค่เรื่อง **touch** เท่านั้น ส่วนเรื่อง **หมุนจอ (rotate)** มีแต่สมุดส่วนตัว (แบบที่ 2) เพียงอย่างเดียว ไม่มีทางเลือกอื่น

นี่คือจุดกำเนิดของปัญหา: ถ้าสมุดส่วนตัวของคนที่ login (เช่น kiosk-user) "หยุดอ่านเงียบๆ" เพราะรอไม่ทันตามข้อด้านบน — **จอจะไม่ถูกหมุนเลย (ค้างที่ Normal)** แต่ **ทัชยังคงถูกตั้งไว้ตาม "ป้ายหน้าบ้าน" ที่บอกว่าให้ตั้งเป็น Left** เพราะป้ายหน้าบ้านทำงานเสร็จไปตั้งแต่ก่อน login แล้ว จึงเกิดอาการ **จอปกติ + ทัชยังคิดว่าจอเอียง = แตะไม่ตรงจุดทันที**

## 3. ตารางสรุปไฟล์ทั้งหมดที่เกี่ยวข้อง

| ไฟล์ | เปรียบเหมือน | เก็บอะไร | ใช้ได้กับใคร | อ่านตอนไหน |
|---|---|---|---|---|
| `/etc/X11/xorg.conf.d/99-vending-touchscreen.conf` | ป้ายหน้าบ้าน | ทิศทาง**ทัชสกรีน**อย่างเดียว | ทุกคนที่ login เข้าเครื่อง | ตอนหน้าจอกราฟิกเริ่มทำงาน (ก่อน login) |
| `~/.xprofile` (ของแต่ละคน) | สมุดส่วนตัว — หน้าปกที่บอกว่า "ให้เปิดสมุดข้อ 2" | คำสั่งเรียกไฟล์ด้านล่างให้ทำงานตอน login | เจ้าของบ้าน (user) คนนั้นคนเดียว | ทันทีที่ user คนนั้น login |
| `~/.config/vending-auto-setup/display-session.sh` (ของแต่ละคน) | สมุดส่วนตัว — เนื้อหาจริง | ทิศทาง**จอ**และทิศทาง**ทัช**พร้อมกัน (ค่าที่ตั้งไว้ล่าสุดตอนกด Apply) | เจ้าของบ้าน (user) คนนั้นคนเดียว | หลัง login รอ 5 วิ แล้วเช็คซ้ำสูงสุด 30 ครั้ง — ถ้าไม่เจอจอ/ทัชเลยจะหยุดเงียบๆ |
| `~/.config/vending-auto-setup/display-session.log` (ของแต่ละคน) | สมุดจดบันทึกว่าสมุดข้างบน "ทำสำเร็จไหม" | log ว่ารอบล่าสุดที่ login เจอจอ/ทัชไหม สำเร็จหรือไม่ | เจ้าของบ้าน (user) คนนั้นคนเดียว | ไม่มีใครอ่าน เป็นไฟล์ไว้ให้ "คนดูแลระบบ" เปิดเช็คเองตอนสงสัยว่าทำไมไม่ apply |

## 4. Timeline: เปิดเครื่อง 1 รอบ เกิดอะไรขึ้นตามลำดับเวลาจริงๆ

```
[เปิดเครื่อง]
     │
     ▼
① หน้าจอกราฟิกเริ่มทำงาน (Xorg)
     → อ่าน "ป้ายหน้าบ้าน" (99-vending-touchscreen.conf) ทันที
     → ตั้งทิศทางทัชสกรีนตามป้ายนั้น (เช่น "Left")
     → ⚠️ การหมุนจอ (rotate) ยังไม่มีอะไรสั่งเลยตอนนี้ — จอยังเป็นทิศทาง Normal ของฮาร์ดแวร์
     │
     ▼
② ระบบ auto-login เข้า user ที่ตั้งไว้ (เช่น kiosk-user)
     │
     ▼
③ สมุดส่วนตัวของ user คนนั้น (.xprofile) ถูกเปิด
     → เรียก display-session.sh ทำงาน "แบบเงียบ" อยู่เบื้องหลัง
     │
     ▼
④ display-session.sh รอ 5 วินาที แล้วเริ่มเช็คซ้ำทุก 1 วิ (สูงสุด 30 ครั้ง)
     ├─ ถ้าเจอจอ+ทัชทันเวลา → สั่งหมุนจอ + ตั้งทัชใหม่ทับป้ายหน้าบ้าน (ค่าจะตรงกันหมด ✅)
     └─ ถ้าเช็คจนครบ 30 ครั้งไม่เจอ → หยุดทำงานเงียบๆ ไม่มีอะไรแจ้งเตือน (❌ จอไม่ถูกหมุน)
     │
     ▼
⑤ ผลลัพธ์ที่เห็นบนจอ
     → กรณี ④ สำเร็จ: จอ+ทัชตรงกันตามที่ตั้งไว้
     → กรณี ④ ล้มเหลว: จอค้างที่ Normal (จากข้อ①) แต่ทัชยังเป็น Left (จากป้ายหน้าบ้านข้อ①) = เพี้ยนทันที
```

**สิ่งที่เจอจริง (Reset → ตั้ง Left → เปิด Persist Xorg → restart) ตรงกับเส้นทาง ④ ล้มเหลว** — ป้ายหน้าบ้านเขียนไว้ถูกแล้ว (เช็คได้จากแท็บ "Config Files" ในภาพที่ 2 ว่ามี Left matrix ถูกต้อง) แต่สมุดส่วนตัวของ user ที่ login ไม่สามารถหมุนจอได้ทันในช่วง 35 วินาทีแรกหลัง login (หรือด้วยเหตุผลอื่นที่ทำให้ script หยุดก่อนถึงขั้นหมุนจอ)

## 5. ทำไม toggle บนหน้าเว็บถึงดูเหมือน "หลอก" เรา

- **"Persist at login"** (ค่า default ติ๊กไว้เสมอ) กับ **"Persist touch in Xorg"** (ค่า default ไม่ติ๊กไว้เสมอ) — 2 ปุ่มนี้**ไม่ได้อ่านสถานะจริงจากเครื่องเลย** ทุกครั้งที่เปิดหน้าเว็บใหม่ มันจะรีเซ็ตกลับไปเป็นค่า default เสมอ **ไม่ว่าจริงๆ แล้วไฟล์นั้นจะถูกเขียนไว้แล้วหรือยัง**
- ต่างจากปุ่มหมุนจอ (Normal/Left/Right) และ "ทิศทาง Touch" ที่**อ่านค่าจริงจากเครื่องมาโชว์**ให้ทุกครั้งที่เปิดหน้า (เพราะทำแบบนี้ตั้งใจไว้ตั้งแต่แรก)
- วิธีเดียวที่จะรู้ว่าไฟล์ไหนถูกเขียนไว้จริง ต้องไปดูที่แท็บ **"Config Files"** ในหน้าเดียวกัน ซึ่งจะโชว์เนื้อไฟล์จริงๆ (ไม่ใช่สถานะ toggle)

## 6. ลำดับที่ควรตั้งค่า — ทำตามนี้ทีละขั้น

### ขั้นที่ 1 — ตั้ง Kiosk Mode ให้เสร็จสมบูรณ์ก่อน (หน้า "คีออส")

ต้องทำให้ครบ **4/4** ก่อนเสมอ ก่อนจะไปยุ่งกับหน้า "จอแสดงผล":

1. สร้าง user สำหรับ kiosk (เช่น `kiosk-user`)
2. ตั้ง Auto-login ให้ user นั้น
3. ติดตั้งซอฟต์แวร์ (Openbox + Chromium)
4. ตั้ง Autostart เบราว์เซอร์ (ใส่ URL ที่จะเปิดตอน login) — **ข้อนี้ในภาพที่ให้มายังไม่เสร็จ ("ยังไม่ครบ") ต้องทำให้เสร็จก่อน**

ทำไมต้องทำก่อน: เพราะขั้นตอนถัดไป (ตั้งจอ/ทัช) ต้องตั้งค่า**ตอนที่ user สำหรับ kiosk เป็นคนล็อกอินอยู่หน้าจอเครื่องจริง** ถ้ายังสลับ user ไม่เสร็จ การตั้งจอ/ทัชจะไปเขียนผิดคน (ไปลง "สมุดส่วนตัว" ของ user คนอื่นแทน)

### ขั้นที่ 2 — Reboot 1 ครั้ง แล้วเช็คว่า user ของ kiosk เป็นคน login อยู่จริง

รอให้เครื่อง auto-login เข้า user kiosk เสร็จ อย่าเพิ่งไปตั้งจอ/ทัชทันที — ให้แน่ใจก่อนว่าตอนนี้คนที่นั่งอยู่หน้าจอ (ในทางเทคนิค) คือ user ของ kiosk แล้วจริงๆ ไม่ใช่ user แอดมินเดิม

### ขั้นที่ 3 — ตั้งค่าจอ + ทัช (หน้า "จอแสดงผล") ทีละอย่าง อย่ารวบทำพร้อมกันทั้งหมด

ทำทีละรอบ แล้ว**เช็คผลก่อนไปขั้นต่อไป**:

**รอบที่ 1**: ตั้งทิศทางจอ (เช่น Left) → ติ๊กแค่ **"Persist at login"** อย่างเดียว (ปล่อย "Persist touch in Xorg" ไว้ก่อน) → กด Apply → **Reboot** → เช็คว่าจอ+ทัชกลับมา Left ถูกต้องหรือไม่หลัง reboot

- ถ้า**ผ่าน**: แปลว่าสมุดส่วนตัว (`.xprofile` + `display-session.sh`) ทำงานได้ปกติ ไปรอบที่ 2 ได้
- ถ้า**ไม่ผ่าน** (จอกลับเป็น Normal): แปลว่าสมุดส่วนตัวรอไม่ทัน หรือหาจอ/ทัชไม่เจอ ต้องไปเปิดไฟล์ log (`display-session.log`) เช็คก่อน **อย่าเพิ่งไปทำรอบที่ 2** เพราะจะยิ่งซ้อนปัญหา

**รอบที่ 2** (ทำเฉพาะเมื่อรอบที่ 1 ผ่านแล้ว): ติ๊กเพิ่ม **"Persist touch in Xorg"** (ตอนนี้ทิศทางจอที่เลือกอยู่ในหน้าเว็บต้องเป็นค่าที่ถูกต้องล่าสุดจากรอบที่ 1 แล้ว — อย่าเพิ่งไปแตะ dropdown เลือกจอ/ทัชก่อนกด เพราะจะรีเซ็ตค่าที่เลือกไว้) → กด Apply → **Reboot** → เช็คอีกครั้งว่าจอ+ทัชยังตรงกันอยู่

### ขั้นที่ 4 — ปิดหน้าจออัตโนมัติ (Screen Blank) — ทำเป็นลำดับสุดท้าย

เป็นการตั้งค่าที่ไม่เกี่ยวกับจอ/ทัชโดยตรง (แค่กำหนดว่ากี่นาทีถึงจะดับจอ) ทำหลังสุดได้ ไม่กระทบกับขั้นตอนข้างบน

## 7. หลัง Reboot แต่ละครั้ง เช็คแบบไหนถึงจะชัวร์สุด

อย่าเช็คแค่ด้วยตา (ดูจอ) เพราะบางทีจอถูกต้องแต่ทัชยังเพี้ยนอยู่แบบมองไม่ออกจนกว่าจะลองแตะ ให้ทำ 2 อย่างนี้:

1. **ลองแตะจอจริงๆ** ที่มุมและขอบต่างๆ ไม่ใช่แค่ดูว่าภาพตรงหรือไม่
2. **เปิดแท็บ "Config Files"** ในหน้า "จอแสดงผล" เช็คว่าไฟล์ `99-vending-touchscreen.conf` กับเนื้อหาสคริปต์ `display-session.sh` มีค่าตรงกัน (ทิศทางเดียวกัน) ไม่ใช่คนละค่า

หากมีสิทธิ์เข้าเครื่องได้ (ผ่านทีมเทคนิค) ให้ขอให้เปิดไฟล์ `display-session.log` ในโฟลเดอร์บ้านของ user kiosk เพื่อดูว่ารอบ boot ล่าสุด สคริปต์หาจอ/ทัชเจอไหม หรือหยุดเงียบๆ ไปกลางทาง — ไฟล์นี้คือหลักฐานที่ชัดที่สุดว่าเกิดอะไรขึ้นจริง

---

## 8. สรุปกฎทองคำ 3 ข้อ

1. **ตั้ง Kiosk Mode ให้ครบ 4/4 และ reboot จนแน่ใจว่า user kiosk login อยู่จริง ก่อนไปยุ่งกับหน้า "จอแสดงผล" เสมอ**
2. **อย่าติ๊ก "Persist at login" กับ "Persist touch in Xorg" พร้อมกันในรอบแรก** — ทดสอบทีละอย่าง reboot เช็คผลก่อน ค่อยเพิ่มอีกอย่าง
3. **toggle บนหน้าเว็บไม่ได้บอกความจริงเสมอไปว่าไฟล์ถูกเขียนไว้หรือยัง** — เชื่อแท็บ "Config Files" กับการลองแตะจอจริงเท่านั้น
